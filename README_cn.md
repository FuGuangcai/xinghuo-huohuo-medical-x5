# 星光火火_RDKX3智慧医疗巡检机器人
队伍名称：星光火火队
院校：阜阳理工学院
负责人：郭逸凡
主控硬件：地平线RDK X3
竞赛平台：OriginCar Pro智能车模

## 一、整体系统架构说明
整机四层分层架构，全程无人工干预，适配赛事180秒计时规则：
1. 底层硬件驱动层：底盘电机、转向舵机、外设供电驱动；
2. 多源感知层：激光雷达避障、MIPI视觉采集、IMU位姿矫正；
3. RDK X3 AI推理层：二维码解码、YOLO人形检测、VLM多模态图文生成；
4. 任务决策状态机层：分阶段管控大厅寻码、诊疗巡检、返程停车三大子任务。

## 二、硬件选型与连接方式
1. 核心主控：RDK X3开发板，所有AI计算本地NPU运行；
2. 车模载体：原厂OriginCar Pro，车体、电机、舵机、车轮不做改动；
3. 感知设备：MIPI相机接入板载MIPI接口，激光雷达USB连接，IMU串口通讯；
4. 输出设备：OLED显示屏、语音扬声器串口接入X3；
5. 供电方案：5000mAh锂电池统一为底盘与开发板供电。

## 三、软件系统设计思路
基于ROS轻量化开发，适配RDK X3嵌入式算力：
1. 驱动模块：电机闭环PID控制，实时接收上层速度、转向指令；
2. 感知融合模块：同步处理雷达障碍物数据、视觉图像帧；
3. AI推理模块：模型提前量化，并行完成目标检测、图文生成，低延迟输出结果；
4. 任务调度模块：有限状态机自动切换3个子任务逻辑，内置计时、罚时统计功能。

## 四、关键任务实现策略
1. 子任务1大厅任务获取：激光动态避障+视觉导航抵达二维码点位，识别后语音+屏幕同步播报巡检方向；
2. 子任务2诊疗区环形巡检：HSV视觉识别黄色通道循迹，检测人形立牌后调用VLM生成文字并播报；
3. 子任务3自主返程：重复大厅避障逻辑，视觉识别P停车区，车身2/3入区判定任务结束。

## 五、竞赛任务及规则适配说明
1. 场地约束：区分大厅、连接通道、诊疗环形通道，程序禁止驶入浅绿色诊疗禁行区域；
2. 罚时规避方案：雷达提前0.8米减速绕行减少碰撞罚时；双通道视觉纠偏防止越界；优化AI推理速度避免未图文播报罚时；
3. 时限约束：软件内置180秒全局倒计时，剩余10秒简化路径保障基础任务完成；
4. 赛事规范：全程自主运行，不人工干预、不冲撞赛道墙体与边界。

## 六、仓库配套资料
1. 程序源码：底盘驱动、感知融合、AI推理、任务调度全套ROS工程；
2. AI文件：适配RDK X3量化RKNN人形检测、多模态模型；
3. 文档资料：硬件接线图纸、赛道调试记录、系统架构说明图；
4. 运行脚本：一键启动全流程自动化巡检脚本。
